# Zynq FIR Decimator 시스템 데이터 파이프라인

## [Phase 1] 시스템 셋업 및 초기화 (Offline / Setup Phase)

### 1. PC **$\rightarrow$** PS (비트스트림 및 펌웨어 다운로드)

* **데이터 흐름:** PC (Vitis/xsdb) **$\rightarrow$** USB-JTAG 케이블 **$\rightarrow$** Zynq Debug Access Port (DAP) **$\rightarrow$** OCM / DDR 메모리
* **사용 모듈 & 프로토콜:**
  * `xsdb` (Xilinx System Debugger)
  * `ps7_init.tcl` (Zynq PS 초기화 스크립트: DDR 컨트롤러 및 PLL 클럭 활성화)
  * **JTAG AXI Master:** JTAG 신호를 AXI 메모리 맵(Memory Map) 쓰기 트랜잭션으로 변환
* **핵심 동작:** FPGA 영역(PL)에 하드웨어 로직(`.bit`)을 굽고, `mwr+mrd` 우회 스크립트를 통해 C 프로그램(`.elf`)을 DDR 메모리에 오염 없이 적재함.

---

## [Phase 2] 런타임 데이터 처리 (Runtime Execution Phase)

### 2. PS 내부 연산 및 메모리 적재 (입력 신호 생성)

* **데이터 흐름:** Cortex-A9 레지스터 **$\rightarrow$** L1/L2 캐시 **$\rightarrow$** DDR 메모리 (`src_buf`)
* **사용 모듈 & 프로토콜:**
  * **ARM Cortex-A9 Core 0:** 베어메탈 C 코드(`gen_multitone`) 실행
  * **DDR Memory Controller (DDRC):** 캐시의 데이터를 실제 물리 RAM에 기록
* **핵심 동작:** 8192개의 멀티톤 입력 샘플을 생성. CPU가 계산한 값이 캐시에만 머물지 않도록 반드시 `Xil_DCacheFlushRange()`를 호출하여 DDR 메모리와 동기화(Flush)함.

### 3. PS **$\rightarrow$** PL 데이터 전송 (DMA 송신)

* **데이터 흐름:** DDR 메모리 (`src_buf`) **$\rightarrow$** Zynq HP/ACP 포트 **$\rightarrow$** AXI DMA (MM2S) **$\rightarrow$** FIR IP (S_AXIS)
* **사용 모듈 & 프로토콜:**
  * **AXI DMA (MM2S 채널):** Memory Map to Stream 변환 엔진
  * **AXI Interconnect:** 여러 AXI 마스터/슬레이브 간의 라우팅
  * **프로토콜 변환:** 주소(Address) 기반의 AXI4 (Memory Map) **$\rightarrow$** 주소 없는 AXI4-Stream (`tdata`, `tvalid`, `tlast`)
* **핵심 동작:** 소프트웨어 개입 없이 DMA가 8192개의 샘플을 100MHz 속도로 하드웨어로 펌핑함. 마지막 8192번째 샘플 송신 시 자동으로 `s_axis_tlast`를 High로 인가.

### 4. PL 내부 하드웨어 신호 처리 (필터링 및 데시메이션)

* **데이터 흐름:** FIR IP 입력 버퍼 (S_AXIS) **$\rightarrow$** Transposed Form 연산 파이프라인 **$\rightarrow$** FIR IP 출력 버퍼 (M_AXIS)
* **사용 모듈 & 프로토콜:**
  * `fir_decimator_n43_axis.v` (AXI-Stream Wrapper): 스키드 버퍼(Skid buffer), 동적 카운터, Auto-Flush 상태 머신
  * `fir_decimator_n43.v` (Core): N=43 MAC 연산기, M=2 위상(Phase) 선택 데시메이터 로직
* **핵심 동작:** 100MHz 클럭(10ns) 단위로 데이터 실시간 연산. `s_axis_tlast` 감지 후 파이프라인에 0을 강제 주입(Auto-Flush)하여 갇힌 데이터를 밀어내고, 정확히 4096개의 결과와 `m_axis_tlast`를 출력함.

### 5. PL **$\rightarrow$** PS 데이터 수신 (DMA 수신)

* **데이터 흐름:** FIR IP (M_AXIS) **$\rightarrow$** AXI DMA (S2MM) **$\rightarrow$** Zynq HP/ACP 포트 **$\rightarrow$** DDR 메모리 (`dst_buf`)
* **사용 모듈 & 프로토콜:**
  * **AXI DMA (S2MM 채널):** Stream to Memory Map 변환 엔진
  * **프로토콜 변환:** AXI4-Stream **$\rightarrow$** AXI4 (Memory Map)
* **핵심 동작:** FIR IP가 뱉어내는 4096개의 결과 샘플을 받아 DDR 메모리에 순차 기록. `m_axis_tlast` 신호를 받으면 해당 패킷 수신을 종료하고 상태 레지스터의 IDLE 비트를 1로 전환함.

### 6. PS **$\rightarrow$** PC 결과 전송 (UART 통신)

* **데이터 흐름:** DDR 메모리 (`dst_buf`) **$\rightarrow$** L1/L2 캐시 **$\rightarrow$** Cortex-A9 **$\rightarrow$** PS UART1 컨트롤러 **$\rightarrow$** USB-UART 칩 **$\rightarrow$** PC
* **사용 모듈 & 프로토콜:**
  * **Zynq PS UART Controller:** 병렬 데이터를 직렬 통신(Serial)으로 변환
  * **UART 프로토콜:** 115200bps, 8N1 (8 Data bits, No parity, 1 Stop bit)
* **핵심 동작:** DDR의 최신 데이터를 읽기 전 `Xil_DCacheInvalidateRange()`로 캐시를 무효화. 이후 4096개의 샘플과 Magic Word(`0xDEADBEEF`)를 바이트 단위로 쪼개어 UART TX FIFO에 밀어 넣음.

### 7. PC 내부 결과 시각화 (Python)

* **데이터 흐름:** PC COM 포트 **$\rightarrow$** Python 버퍼 **$\rightarrow$** Numpy (FFT 연산) **$\rightarrow$** Matplotlib 화면 출력
* **사용 모듈 & 패키지:**
  * `pyserial` (직렬 포트 수신)
  * `numpy` (고속 푸리에 변환 및 배열 처리)
  * `matplotlib.pyplot` (주파수/시간 도메인 그래프 시각화)
* **핵심 동작:** 직렬 포트로 수신된 바이트 스트림을 조립하여 4096개의 정수 배열로 복원하고, 시나리오(15MHz 통과, 25MHz 차단)에 맞게 FFT 스펙트럼을 화면에 렌더링함.
