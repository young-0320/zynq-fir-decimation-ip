## A. Ⅰ. 도전결과 및 성과 — 초안 전문 (붙여넣기용)

### 1. 총평

본 도전의 목표는 1D FIR Decimation 필터를 대상으로 사양 확정부터 골든모델 구축, RTL 설계, 시뮬레이션 검증, FPGA 실보드 실증까지를 1인 독립 수행으로 완주하는 것이었음. 결과적으로 계획한 5단계를 모두 달성하였을 뿐 아니라, 지도교수님의 피드백을 반영하는 과정에서 당초 계획을 상회하는 수준으로 범위가 확장되었음. 구체적으로는 단순 RTL 검증을 넘어 Zynq PS-PL을 AXI DMA로 연결한 end-to-end 실시간 데모 시스템을 구축하였고, CPU 대비 처리시간 벤치마크, 최대 동작 주파수(Fmax) 스윕, ASIC 합성 플로우 교차 검증, 보드 전력 실측까지 다면적 보조지표를 확보하였음. 최종 산출물은 실보드에서 골든모델 대비 SNR 74.9dB 수준으로 일치가 확인된 검증 완료 IP이며, 전 과정이 문서와 스크립트로 기록되어 제3자가 재현 가능한 상태로 정리되어 있음.

#### 2-2. 계획 대비 변경 내용

**첫째, 설계 사양의 정밀화.** 계획 단계에서는 제출용 탭 수 N=41, 고정소수점 포맷 Q4.12를 상정하였으나, 실제 계수 설계 결과 목표 감쇠 60dB를 만족하는 Kaiser window 설계에서 **N=43**으로 확정하였고, 계수의 동적 범위를 분석한 결과 정수부 여유가 불필요하여 소수부 정밀도를 극대화하는 **Q1.15 signed 16-bit**로 변경하였음. 이는 골든모델 기반 정량 분석이 계획 단계의 가정을 데이터로 교정한 사례임.

**둘째, 시연 구조의 전면 확장.** 당초 실증은 BRAM에 저장된 데이터를 재생하여 확인하는 수준을 상정하였으나, "시연에서 보여지는 것이 중요하다"는 지도교수님 피드백을 반영하여, PC(Python) → UART → PS bare-metal C → AXI DMA → PL FIR/데시메이터 → DMA → UART → PC FFT 시각화·자동 판정으로 이어지는 **end-to-end 실시간 데모 시스템**으로 확장하였음. 이 과정에서 AXI-Stream 래퍼, AXI DMA 연동, SD boot, bare-metal 펌웨어, PC 자동화 도구가 추가 개발되었음.

**셋째, 보조지표의 추가.** 4차 지도미팅에서 승인받은 방향에 따라 계획에 없던 세 가지 보조지표 — CPU 대비 처리시간 벤치마크, ASIC 합성 플로우 기반 면적·타이밍·전력 분석, 보드 전력 실측 — 를 추가로 수행하여, 단일 기능 구현을 넘어 PPA(성능·전력·면적) 관점의 다면 평가 체계를 갖추었음.

### 3. 도전 성과

#### 3-1. 실보드 기능 검증 — 정량 판정 PASS

보드 출력 4,096 샘플을 Python 고정소수점 골든모델과 직접 비교하는 자동 판정 체계를 구축하였고, 두 검증 시나리오 모두 통과하였음.

| 시나리오 (입력 톤) | Max error | RMSE | SNR | Correlation | 판정 |
| --- | :--: | :--: | :--: | :--: | :--: |
| 1-1 (5/20/30MHz) | 6 LSB | 1.40 LSB | 74.86dB | 1.000000 | PASS |
| 1-2 (7/15/25/45MHz) | 7 LSB | 1.81 LSB | 72.22dB | 1.000000 | PASS |

특히 시나리오 1-2의 45MHz 톤은 데시메이션 후 5MHz로 alias될 수 있는 성분인데, 설계한 필터가 이를 약 64dB 수준으로 억제함을 실보드에서 확인하여 anti-aliasing 필터의 핵심 기능을 정량적으로 입증하였음.

#### 3-2. CPU 대비 성능 — FPGA 약 2.6배, 이론 한계 대비 오차 약 3.8%

동일한 FIR 연산(N=43, 8,192샘플)을 노트북 CPU(numpy float64)와 FPGA(Q1.15)에서 각각 측정하였음. UART 전송은 관측 경로이므로 제외하고, CPU는 연산 구간만, FPGA는 DMA 전송 구간만 측정하는 엄밀한 기준을 적용하였음.

| 환경 | CPU | FPGA | 비고 |
| --- | :--: | :--: | --- |
| Windows(발표 환경, i5-1340P) | 221.0µs | 85.0µs | FPGA 약 2.6배 우위 |
| 이론 처리 시간 | — | 81.92µs | 측정값과 오차 약 3.8% |

FPGA 측정값 85.0µs는 이론 처리 시간(8192÷2÷100MHz = 81.92µs)과 약 3.8% 오차로 일치하여, 구현이 이론 한계에 근접해 동작함을 보였음. 아울러 별도 고성능 데스크톱 환경(Ryzen 7 9700X, Ubuntu, BLAS 최적화 활성)에서는 CPU가 58.6µs로 FPGA보다 빠를 수 있음을 함께 제시하였음. 즉 CPU의 처리 시간은 하드웨어 사양과 소프트웨어 스택에 따라 수 배 범위로 변동하는 반면 FPGA는 실행 환경과 무관하게 일정하며, FPGA의 본질적 이점이 절대 속도가 아니라 결정론적 처리 시간에 있음을 정직하게 분석하였음. 측정 조건(CPU 스펙·방법론)은 교수님 지도에 따라 비교 차트에 직접 명기하였음.

> 수치 히스토리: 4차 미팅(7/2) 보고 시점에는 162.0µs/83.0µs(1.95배)였으나 이후 반복
> 재측정에서 221.0µs/85.0µs로 정착 (`docs/log/49`). Ⅲ장 4-4는 미팅 당시 보고 수치를
> 역사 기록으로 유지하고 정정 문장을 부기.

#### 3-3. 타이밍 여유(Fmax) 측정과 구조 개선 — Fmax 116→146MHz(+26%)

Vivado 주파수 스윕으로 초기 구조(v1)의 Fmax가 116MHz임을 확정한 뒤, 타이밍 위반이 발생하는 지점의 크리티컬 패스를 분석하여 병목이 누산 가산과 라운딩이 한 사이클에 겹치는 경로(FPGA CARRY4 캐리 체인)임을 규명하였음. 라운딩을 별도 파이프라인 스테이지로 분리한 개선 구조(v2)를 설계하여 **Fmax를 146MHz로 26% 향상**시켰고, v1@115MHz와 v2@145MHz 모두 실보드에서 SD boot 데모 정상 동작을 확인하였음. 자원은 코어 기준 DSP48 16개, LUT 1,792개, FF 2,113개로 두 구조가 동급이며, 주파수 향상이 자원 증가 없이 구조 개선만으로 달성되었음.

#### 3-4. ASIC 합성 교차 검증 — 최적화 이득의 타겟 종속성 실증

교수님 지시에 따라 학교 서버의 ASIC 합성 플로우(Oasys-RTL, 250nm 표준셀)로 v1/v2 두 구조를 동일 제약 하에 스윕(주기 20,000→6,000ps)하여 면적·타이밍·전력을 분석하였음. 그 결과 250nm 공정임에도 두 구조 모두 166.7MHz 이상에서 타이밍을 만족하였고, FPGA에서 26% 격차를 보였던 두 구조가 **표준셀 합성에서는 동률로 수렴**함을 확인하였음. 동일한 병목 경로를 FPGA CARRY4는 8.664ns에 처리한 반면 표준셀 합성기는 가산기 재구조화로 5.719ns에 흡수한 것이 직접 원인으로, **v2 구조 개선의 이득이 FPGA의 고정 캐리 체인이라는 물리 구조에 종속된 타겟 특화 최적화임을 데이터로 실증**하였음. 이는 단일 RTL을 두 구현 타겟에서 교차 조명한 비교 실험으로, 후속 P&R은 서버 툴 내부 오류로 중단되었으나 원인과 재개 절차를 문서화하여 한계를 명확히 구분해 기록하였음.

#### 3-5. 보드 전력 실측 — Vivado 추정 대비 정합 확인

교수님께서 대여해 주신 HDS242 계측기를 보드 5V 전원 입력에 직렬 삽입하여, 세 가지 상태의 전류·전압을 실측하였음(상태당 전류 5회 판독 중앙값, 전압은 부하 상태에서 상태별 실측). 측정 결과 S0(PL 미구성) 1.72W, S1(boot 완료 idle) 2.21W, S2(데모 실행 중) 2.18W로, 상태 차분 S1−S0 = 0.49W가 FPGA 구성·클럭·앱 구동·DDR 활성이 추가로 소비하는 몫으로 분리되었음. S2와 S1의 차이는 노이즈 수준으로, FIR 연산 버스트(85µs)가 평균 전력에 잡히지 않을 만큼 짧다는 사전 예측이 실측으로 확인되었음. 보드 실측(2.21W)이 Vivado on-chip 추정(1.705W)보다 큰 것은 추정에 포함되지 않는 보드 주변부가 더해지기 때문이며, 측정 과정에서 USB-UART 관측 인프라의 소비(0.22W)를 실측으로 분리하고 레귤레이터 변환 손실을 감안하면 두 값은 정합 범위에 있음을 확인하였음. 아울러 관측용 UART 인프라가 FIR 연산 자체보다 큰 전력을 소비한다는 점은 "UART는 관측 경로"라는 본 설계의 구조적 구분을 전력 데이터로도 뒷받침함.

#### 3-6. 검증 방법론 — 재현·규명·회귀의 체계화

본 도전에서 가장 큰 성장은 문제를 다루는 방법의 체계화였음. 대표 사례 두 가지를 기록함.

**사례 1 — DMA 전송 실패의 근본 원인 규명.** 보드 bring-up 중 모든 경로에서 DMA 전송이 멈추는 문제가 발생하였음. FIR를 제거한 최소 구성(smoke test)과 디버그 경로를 별도로 만들어 문제 위치를 이분 탐색한 결과, 원인은 RTL이 아니라 전송량 16,384바이트가 AXI DMA 기본 길이 필드(14-bit, 최대 16,383바이트)를 **정확히 1바이트 초과**한 설정 문제임을 규명하였고, 길이 필드 확장으로 전 경로를 복구하였음.

**사례 2 — 인터페이스 데드락의 구조적 해결.** 시뮬레이션을 통과한 AXI-Stream 래퍼에서 코드 리뷰 중 잔존 데드락 가능성을 발견하였음. 입력 버블이 임계값 이상이면 마지막 출력이 프레임 종료 신호 없이 나가 시스템이 영구 정지하는 조건이었음. 수정에 앞서 **새 테스트벤치를 수정 전 RTL에 먼저 적용하여 예측한 임계값과 정확히 일치하는 실패를 재현**함으로써 테스트 자체의 유효성을 증명한 뒤, 출력을 프레임 종료 확정 시까지 보류하는 구조적 수정(hold-back)을 적용하고 버블 스윕 회귀 시험 전체 통과로 마무리하였음.

이러한 과정을 통해 "골든모델 → RTL 비트정확 비교 → 인터페이스 강건성 시험 → 보드 자동 판정"으로 이어지는 계층적 검증 파이프라인이 완성되었으며, 이는 본 도전이 남긴 가장 재사용 가치가 높은 자산임.

### 4. 향후 대학 및 사회에 기여할 방안

**첫째, 재현 가능한 공개 학습 자산화.** RTL 소스, Python 골든모델, 검증 스크립트, 재현 절차를 GitHub에 공개하고, 설계 의도와 오류 원인·해결 과정을 담은 기술 포스팅을 Velog에 연재(3편 이상)하며, 실보드 데모 영상을 YouTube에 업로드할 계획임. 단순 코드 공개가 아니라 '검증 환경 자체'를 재현할 수 있도록 문서화하는 것이 목표이며, 최종 발표(8/7) 이전까지 순차 완료할 예정임.

**둘째, 전공 후배의 진입 장벽 완화.** 본 도전은 FPGA 수업 수강에 실패한 학생이 수업 밖에서 독립적으로 실보드 검증까지 도달한 과정 그 자체이므로, 같은 상황의 후배들이 동일 보드와 절차로 따라올 수 있는 튜토리얼로서 실질적 가치가 있음. 특히 DMA 설정 한계, 인터페이스 데드락 등 실제 하드웨어에서만 만나는 문제들의 원인 분석 기록은 후배들의 시행착오를 단축시킬 것임.

**셋째, 지속적 관리와 확장.** 공개 자산을 일회성으로 끝내지 않고, 본 IP를 기반으로 한 검증 방법론 확장(어서션·커버리지 추가 등)을 지속하여, 설계뿐 아니라 검증까지 학습할 수 있는 자산으로 유지·발전시키겠음.
