# FIR Decimation 프로젝트 워크플로우 v2

- 작성일: 2026-03-11
- 목적: 입력 신호 생성 제약과 멀티톤 자극을 재현 가능하게 확정

## 1) 완료 조건 (Definition of Done)

1. `docs/input_signal_spec.md`에 입력 신호 계약이 정리되어 있음.
2. 멀티톤 톤 개수, 주파수 조합, 진폭 배분, phase 정책, headroom 규칙이 문서에 명시되어 있음.
3. 입력 신호 생성이 스크립트/함수/파라미터 기준으로 재현 가능하게 정리되어 있음.
4. ideal 모델 검증과 이후 fixed Q-format 결정에 필요한 동적 범위 가정이 문서에 남아 있음.
5. 이후 fixed/golden 단계(v3)로 넘어가기 위한 입력 자극 기준선이 정리되어 있음.

## 2) 루틴

1. 수: 멀티톤 후보와 alias 관찰 목적의 주파수 조합 정리.
2. 목: 진폭 배분, peak/RMS, headroom, phase 정책 확정+로그 정리(`docs/log/*.md`)

## 3) 하루 운영 규칙

1. 하루 마감 10분 동안 아래 5줄을 `docs/log`에 전사:
   - 오늘 결정 1개
   - 근거 수치 1개
   - 실패/수정 1개
   - 재현 명령어 1줄
   - 다음 액션 1개

## 4) 산출물 위치 규칙

1. 스펙/의사결정: `docs/input_signal_spec.md`, `docs/log/*.md`
2. ideal 기준 모델: `model/ideal/*.py`
3. 입력 신호 생성/실험 보조 스크립트: `sim/python/*.py`
4. 테스트: 필요 시 `sim/python/test/*.py`
5. 이미지/그래프: `docs/asset/*`

## 5) 의사결정 기록 템플릿

1. 문제 정의
2. 제약 조건(주파수/진폭/headroom/재현성)
3. 대안(A/B/C)
4. 선택안
5. 근거 데이터(숫자/리포트)
6. 트레이드오프와 후속 계획

## 6) 품질 게이트 (각 단계 공통)

1. 입력 신호 생성 규칙이 재현 가능하게 명시됨
2. 톤 개수/주파수/진폭/phase/headroom이 문서에 남아 있음
3. ideal 검증과 fixed 단계에 넘길 기준선이 정리됨
4. 결과 수치가 문서에 기록됨
5. 재현 명령어가 문서에 존재함

## 7) 업데이트 정책

1. `workflow_v2.md`는 주 1회만 수정(운영 흔들림 방지).
2. 실험/결정 기록은 `docs/log`에 매일 누적.
3. v2의 스코프는 입력 신호 확정 단계까지로 한정한다.
4. fixed/golden 단계부터는 `workflow_v3.md`를 기준 문서로 사용한다.
