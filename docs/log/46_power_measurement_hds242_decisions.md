# 46. 전력 실측(Track B) 측정 방식 의사결정 (HDS242)

작성일: 2026-07-03
상태: 실측 완료 (2026-07-22, §5) — 비교분석은 `../report/fir_n43/summary/power_board_vs_vivado.md`.
준비물·결선·실행 절차는 `docs/workflow/workflow_v23.md` Track B 참고.

## 1. B1 — 측정 지점: 보드 5V 입력 전체 + 상태 차분

검토한 대안 3개:

1. 5V 입력 전체 단순 측정 — 안전·비파괴이나, DDR3/PHY/USB-UART/레귤레이터 손실이
   포함된 절대값 하나만 남아 Vivado on-chip 추정과의 비교분석이 빈약.
2. PL/코어 rail(1.0V VCCINT 등) 직접 측정 — 추정과 가장 직접 비교되나, Zybo Z7-20에는
   전류 션트/측정 헤더가 없어 트레이스 컷/인덕터 제거가 필요 → 대여·실습 보드에
   파괴적이라 **배제**.
3. **[채택]** 1안 + 상태 차분(S0 PL 미구성 / S1 boot 후 idle / S2 `mode 1-1` 실행 중)
   — 비파괴를 유지하면서 차분으로 변수를 분리.

부가 확정 사항:

- Zybo Z7-20 전원 입력은 **5V**(USB 또는 배럴잭). 기존 workflow 문서의 "12V" 표기는
  오기로 확인, 정정함.
- 차분 해석 한계(의사결정 시 인지): S1−S0는 PL 단독이 아니라 PL 구성+클럭 + PS 앱 구동
  + DDR 활성의 합. S2의 FIR+DMA burst는 µs 스케일(8192샘플 @145MHz)이라 DMM 평균에서
  S2 ≈ S1로 나올 가능성이 높음 — "burst가 평균 전력에 잡히지 않을 만큼 짧다" 자체를
  결과로 기록하기로 함.

## 2. B2 — 측정 대상: v2@145MHz (수정 RTL v22 완료본)

- **[채택]** `build/fir_n43_v2_freq_145mhz/output/BOOT.bin`, 시나리오 = `mode 1-1`
  one-shot 반복.
- 근거: 수정 RTL 반영본이라 "v22 수정본으로 실측" 전제와 정합하고, v1↔v2 교체 결정에서
  v2가 유력한 이상 대표 산출물로 측정하는 것이 맞음.
- v1@115는 제외 — Vivado 기준 v2@145와 전력차 6mW(1.699 vs 1.705W)로 보드 레벨에서
  검출 불가, 별도 측정 의미 없음.
- **비교 기준: 1.705W** (`sweep_summary_v2.md` 145MHz, clk_wiz/MMCM 빌드).
  구 90~120MHz 스윕 1.564~1.576W는 PS7 PLL 계열(MMCM 없음)로 약 0.13W 낮음.
- 옵션: 현장 여유 시 100MHz 골든 SD로 한 점 추가 — 구 스윕 1.567W와 직접 비교 +
  MMCM 유무 차이(+0.13W ≈ 26mA@5V)가 보드 레벨에서 검출되는지 확인.

## 3. B3 — 장비 경로: HDS242 DMM 직렬 + 희생 USB 케이블

- HDS242 대여 완료(스코프 프로브·악어케이블 보유).
- 스코프 프로브는 직렬 전류 측정에 사용 불가 → **DMM 리드 + 10A 전용 잭**이 측정 경로.
  보드 예상 전류 0.4~0.7A로 mA 레인지는 퓨즈 위험이 있어 배제.
- 5V 직렬 삽입은 **희생 micro-USB 케이블 VBUS 절단** 방식 채택 — 추가 구매 없이 가능하고
  D+/D−/GND를 유지하면 한 케이블로 UART+전원 겸용 유지. 배럴잭 breakout(+5V 어댑터,
  전원 점퍼 WALL)은 무개조 대안이나 구매 필요.
- 외부 션트 저항·브레드보드 불필요 — DMM 내부 션트 사용. 스코프+션트로 DMA burst 순간
  전류 파형을 찍는 확장은 이번 범위 제외(필요 시 후속).

## 4. 분석 방법 (확정)

- **절대 비교**: P_board(S1) vs Vivado on-chip 1.705W. 실측은 보드 전체 전력(주변부 +
  레귤레이터 효율 ~85~90% 포함)이므로 P_board > 추정이 정상 — 차이 요인 분해를 결과에 명시.
- **차분**: S1 − S0 (PL 단독 아님을 명시).

## 5. 실측 결과 (2026-07-22)

### 5.1 경로 변경: 희생 USB 케이블 → 배럴잭 breakout

§3에서 채택했던 "희생 micro-USB VBUS 절단" 대신 **무개조 대안(배럴잭 breakout)으로 변경**.
5V/2A center-positive(5.5/2.1mm) 어댑터 + DC잭 암수 피그테일을 구매(장학금 잔여 예산)해
케이블 훼손 없이 절단점을 확보했다. 전원 점퍼 WALL, micro-USB는 UART 전용으로 유지.

결선 — 전류(직렬)와 전압(브리지 후 병렬)의 2단 구성:

```
[전류]  어댑터 ─[암]─ +5V ─▶ 적색[10A잭] ─ HDS242(DC A) ─ [COM]흑색 ─▶ +5V ─[수]─ 보드
                      GND ──────────────────── 직결 ──────────────────── GND

[전압]  어댑터 ─[암]─ +5V ──●(직결 브리지)●── +5V ─[수]─ 보드   ● 매듭 = 측정점
                      GND ──●(직결)●────────── GND
        → 적색[VΩ잭]·흑색[COM]을 +5V/GND 매듭에 병렬 접촉 (DC V, 부하 중)
```

- 전환 규칙: 전류→전압 시 적색 리드 **10A→VΩ 잭 이동 필수**(10A 잭은 저임피던스 션트 —
  그 상태로 전압원 병렬 접촉 = 단락), 결선 변경은 어댑터 분리 후.
- 상세 결선도·원리: `../report/fir_n43/summary/power_board_vs_vivado.md` 부록 B.

### 5.2 측정 조건

- 대상: v2@145MHz BOOT.bin (`build/fir_n43_v2_freq_145mhz/output/`), 시나리오 `mode 1-1`
- **전 상태 USB-UART(PC) 연결 기준으로 통일** — S2가 UART 명령을 요구하므로. USB 유무
  오프셋은 부가 측정으로 분리 (§5.4)
- 전류: 상태당 5회 판독 중앙값. 전압: 부하 상태에서 상태별 1회(절단점 다리 연결 후
  DMM 전압 모드로 순차 측정)
- §2의 옵션(100MHz 골든 SD 한 점 추가)은 미실시

### 5.3 측정 표

| 상태 | 판독 I₁~I₅ [A] (시간 순) | I_med [A] | V [V] | P = V×I_med [W] |
| ---- | -------- | --------: | ----: | --------------: |
| S0 (SD 미삽입, PL 미구성) | 0.336 / 0.334 / 0.340 / 0.342 / 0.341 | 0.340 | 5.047 | **1.72** |
| S1 (SD boot, `READY FIR` idle) | 0.440 / 0.443 / 0.451 / 0.440 / 0.443 | 0.443 | 4.988 | **2.21** |
| S2 (`mode 1-1` 반복 실행 중) | 0.434 / 0.434 / 0.456 / 0.435 / 0.438 | 0.435 | 5.008 | **2.18** |

- **S1 − S0 = 0.103A / 0.49W** — PL 구성 + 145MHz 클럭(MMCM) + 앱 구동 + DDR 활성의
  합 (§1의 차분 한계대로 PL 단독 아님)
- **S2 − S1 = −0.03W ≈ 0 (노이즈 범위)** — §1 예상대로 µs 스케일 burst는 DMM 평균에
  미검출. "burst가 평균 전력에 잡히지 않을 만큼 짧다"를 결과로 확정
- 어댑터 무부하 5.183V → 부하 시 4.99~5.05V (전류 증가에 따른 정상 강하)

### 5.4 부가 관찰

- **USB-UART 연결 오프셋**: USB 미연결 판독(시간 순) — S0 0.293 / 0.291 / 0.294 /
  0.295 / 0.292 (I_med 0.293A), S1 0.394 / 0.404 / 0.398 / 0.398 / 0.392 (I_med 0.398A)
  → 연결 시 +47/+45mA ≈ **+0.22W** (FTDI 브리지 활성화 몫). 관측 인프라가 FIR 연산보다
  큰 전력을 소비 — "UART는 관측 경로" 구분을 전력 데이터로도 뒷받침
- USB 유무 양 조건에서 S1−S0 델타 재현(0.105 vs 0.103A) → **차분 방법론 유효성 확인**
- 전원 인가 직후 ~0.30A 과도 후 안정화 (판독은 안정화 후)
- S2 중 순간 0.456A 관측 — UART 전송 구간 활동으로 추정 (중앙값이 이상치 흡수)
- `mode 1-1` reset 없이 반복 실행 정상 동작 — 반복 실행 경로의 보드 확인 겸함

### 5.5 Vivado 추정 대비 (요약)

보드 실측 S1 2.21W vs Vivado on-chip 1.705W (§2 기준). 실측에서 USB-UART 0.22W를
분리하면 1.99W, 레귤레이터 효율 85~90% 가정 시 실리콘+주변부 ≈ 1.69~1.79W로 추정과
정합 범위. 상세 분해·한계는 `docs/report/fir_n43/summary/power_board_vs_vivado.md`.
