# 04. 입력 신호 Q-format 확정

- 작성일: 2026-03-11
- 단계: 1
- fixed/golden input sample format 결정

## 1) 이번 세션 핵심 결정

1. 입력 신호 포맷을 `signed 16-bit, Q1.15`로 확정했다.
2. FIR 계수 포맷과 입력 포맷을 동일하게 `Q1.15`로 맞추기로 했다.
3. 내부 곱셈 결과 포맷은 `signed 32-bit, Q2.30`으로 해석한다.
4. 입력 포맷 결정 근거는 coefficient 호환성, 멀티톤 생성 편의성, 16-bit ADC 확장성, 양자화 SNR, DSP 관행이다.

## 2) 확정 포맷

- 입력 신호: `signed 16-bit, Q1.15`
- FIR 계수: `signed 16-bit, Q1.15`
- 곱셈 결과: `signed 32-bit, Q2.30`
- 표현 범위: `-1.0 ~ +0.999969482421875`
- 해상도: `2^-15 ~= 0.0000305`

## 3) 결정 근거

### 1. 필터 계수와 자연스럽게 호환된다

- 현재 FIR coefficient는 `Q1.15`로 확정되어 있다.
- coefficient 최대값은 `0.400133`, 최소값은 `-0.068268` 수준으로 `Q1.15` 범위 안에 충분히 들어간다.
- 입력 `Q1.15`와 계수 `Q1.15`를 곱하면 내부 곱셈 결과를 `Q2.30`으로 일관되게 해석할 수 있다.
- 따라서 product scaling, truncate 위치, RTL 문서화가 대칭적으로 정리된다.

### 2. 기본 데모 신호 설계가 쉽다

- `numpy`로 생성한 멀티톤을 `float64`로 합산한 뒤, 추가 정규화 없이 바로 `Q1.15`로 변환할 수 있다.
- 예시 변환식:

```python
scaled = signal * (2**15)
rounded = np.where(
    scaled >= 0.0,
    np.floor(scaled + 0.5),
    np.ceil(scaled - 0.5),
)
x_fixed = np.clip(rounded, -32768, 32767).astype(np.int16)
```

- 단, `+1.0`은 `Q1.15`에서 정확히 표현되지 않으므로 상한은 `32767`로 saturation한다.

### 3. 확장 데모 ADC와 호환된다

- 확장 데모에서 `PMOD AD1` 16-bit ADC를 사용할 경우, 입력을 정규화하고 오프셋 제거 후 `[-1.0, 1.0)` 기준으로 매핑하면 `Q1.15`로 바로 연결할 수 있다.
- 따라서 Python 모델, RTL 인터페이스, 향후 보드 데모 경로를 같은 포맷으로 유지할 수 있다.

### 4. 16-bit 양자화 SNR이 현재 스펙을 충분히 만족한다

- 16-bit 기준 양자화 SNR은 약 `96 dB`로 본다.
- 현재 목표 저지대역 감쇠는 `As = 60 dB`.
- 즉 `96 dB > 60 dB`이므로 입력 양자화 정밀도는 현재 필터 스펙 대비 충분하다.

### 5. 일반적인 DSP fixed-point 관행과도 맞는다

- 오디오/통신 계열의 `signed 16-bit` 정규화 신호 표현에서 `Q1.15`는 가장 흔한 기준 중 하나다.
- 별도 정수 비트가 필요하지 않은 normalized signal에서는 `Q1.15`가 가장 단순하다.

## 4) 최종 확정

- 입력 신호: `signed 16-bit, Q1.15`
- FIR 계수: `signed 16-bit, Q1.15`
- 곱셈 결과: `signed 32-bit, Q2.30`
