# docs/log — 개발 기록 원본

이 폴더는 개발 당시의 의사결정·디버깅 기록 원본이다. 사양 결정부터 RTL 설계, 보드
bring-up, 버그 수정, 실측까지의 과정을 시간 순서대로 담는다. 번호가 작성 순서다.

기록은 AI 협업 워크플로우로 작성·정리되었고, 역할 분담은 프로젝트 내내 일정했다.
무엇을 검증할지와 PASS 기준, 설계 대안 중 무엇을 채택할지의 결정, 그리고 보드·계측기·
학교 서버에서의 모든 물리 실행과 결과 확인은 저자의 몫이었다. AI는 그 결정을 구현하고
기록으로 정리하는 협업 도구로 썼다. 본문에서 3인칭 "사용자"로 지칭되는 행위자가 저자
본인이며, 이 분담은 계획 문서의 항목별 [사용자]/[에이전트] 표기로도 남아 있다
(`../workflow/`).

사후에 문체를 다듬거나 내용을 정리하지 않고 그대로 보존한다. 실패한 시도, 중간에
뒤집힌 해석, 되돌린 결정도 남아 있는데, 이는 결과만 보아서는 알 수 없는 판단 과정을
재현 가능하게 남기기 위한 의도적인 선택이다.

정리된 결과만 필요하면 다음을 보면 된다.

- 보드 검증·비교분석 요약: `../report/fir_n43/summary/`
- FPGA Fmax 스윕: `../../vivado/reports/sweep_summary.md`, `sweep_summary_v2.md`
- ASIC 합성 스윕: `../../asic/oasys/results/sweep_report.md`
- 계획 문서(준비물·실행 절차): `../workflow/`
