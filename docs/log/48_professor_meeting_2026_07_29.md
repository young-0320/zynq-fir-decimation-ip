### 0-2. 전력 실측 결과 제시

```text
측정은 보드 5V 전원 입력에 멀티미터를 직렬로 넣어 전류를 재고,
세 가지 상태의 차분으로 변수를 분리했습니다.

  S0 (FPGA 미구성 baseline):  1.72 W
  S1 (부팅 완료, FIR idle):   2.21 W
  S2 (mode 1-1 실행 중):      2.18 W

핵심 결과 세 가지입니다.

첫째, S1 − S0 = 0.49 W가 FPGA 구성 + 145MHz 클럭 + 앱 + DDR 활성의 몫입니다.

둘째, S2가 S1과 같습니다. FIR 연산이 83µs 만에 끝나기 때문에
1~2초 데모 주기에서 평균 전력에 잡히지 않는 것이고,
"연산이 그만큼 짧고 효율적이다"라는 것 자체가 측정 결과입니다.

셋째, Vivado 추정 1.705W와의 비교입니다. 실측 2.21W에서
USB-UART 칩 몫 0.22W를 실측으로 분리하고 레귤레이터 효율 85~90%를 감안하면
1.7W 안팎으로, 추정과 정합하는 범위입니다.
```

### 0-3. ASIC 결과 제시

```text
Oasys로 v1과 v2를 같은 제약으로 20000ps부터 6000ps까지 sweep했습니다.

결과가 흥미로운데, FPGA에서 26% 차이가 났던 v1/v2가
표준셀 합성에서는 전 구간 동률입니다. 둘 다 166.7MHz 이상 통과합니다.

원인까지 확인했습니다. v1의 병목이었던 누산+라운딩 병합 경로를
FPGA CARRY4는 8.664ns에 처리했지만, 표준셀 합성기는
가산기를 재구조화해서 5.719ns에 흡수합니다.
즉 v2 분할의 이득은 FPGA의 고정 캐리 체인이라는 물리 구조에 종속된
타겟 특화 최적화라는 걸 데이터로 증명한 셈입니다.

P&R은 Nitro placer가 내부 오류(SDA101)로 두 번 같은 지점에서 죽어서
툴 버그로 결론 내리고 중단했습니다. 비교 결론은 합성 결과로 완결이라
P&R이 더해줄 것은 "post-route에서도 유지된다" 한 줄뿐이었습니다.
절차는 재개 가능하게 보존해 두었습니다.
```

### 0-4. 세게 물으실 때 짧은 답변

**mode를 돌려도 전력이 안 늘면 측정이 잘못된 것 아닌가?**

```text
아닙니다. 사전에 예상하고 문서화해 둔 결과입니다.
8192샘플 처리가 83µs라, 1~2초 데모 주기에서 duty가 0.01% 수준입니다.
멀티미터는 수백 ms 평균을 표시하므로 원리적으로 안 잡힙니다.
순간 전류 파형을 보려면 스코프+션트 구성이 필요한데,
비파괴 원칙과 범위 관리 차원에서 이번엔 제외했습니다.
```

**PL만의 전력은 분리 못 하나?**

```text
Zybo Z7-20은 코어 레일에 전류 측정 헤더가 없어서
rail 직접 측정은 트레이스 컷이 필요한 파괴적 방법입니다.
대여·실습 보드라 배제했고, 대신 상태 차분으로
"FPGA 구성+클럭이 더해지는 몫 0.49W"까지는 비파괴로 분리했습니다.
Vivado 계층 분해로는 코어 자체는 0.015W 수준입니다.
```

**250nm ASIC이 28nm FPGA보다 빠르다는 게 말이 되나?**

```text
FPGA의 로직은 LUT와 고정 배선을 거치는 오버헤드가 커서,
같은 RTL이라도 표준셀 직접 구현이 유리한 경우가 많습니다.
특히 이 설계의 병목인 캐리 체인은 FPGA에서 재구조화가 안 되는 반면
표준셀 합성은 자유롭게 최적화합니다. 그 차이가 그대로 나타난 것입니다.
```

**ASIC 전력(1.3W)이 FPGA 추정(1.7W)과 비슷한데 비교 의미는?**

```text
두 수치는 성격이 달라 절대 비교하지 않았습니다.
FPGA 1.7W는 PS7 포함 전체 칩, 코어만은 0.015W이고,
ASIC 1.3W는 250nm 공정에서 toggle-rate 가정으로 추정한 코어 값입니다.
비교 목적은 절대치가 아니라 v1↔v2의 상대 비교와 타겟 종속성 확인입니다.
```

### 0-5. v1↔v2 결정 여쭤보기

```text
마지막으로 v1과 v2 중 대표 산출물 결정을 확인받고 싶습니다.

저는 v2를 대표로 하는 게 맞다고 생각합니다.
근거는, Fmax가 116 대 146으로 26% 높고, 자원은 동급이며,
전력 차이는 6mW로 보드에서 검출 불가능한 수준이고,
145MHz 보드 기능 동작과 전력 실측까지 v2 빌드로 완료했기 때문입니다.
ASIC에서 동률이라는 결과도, v2로 가서 손해 볼 것이 없다는 뜻입니다.

v1은 버리는 게 아니라 "타겟 특화 최적화의 비교 사례"로
발표와 문서에 계속 사용합니다.
```
