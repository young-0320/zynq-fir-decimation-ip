## 5. 입력/데이터 Q-format과의 관계

### 5.1 확정 포맷

- 입력 신호 포맷: `signed 16-bit, Q1.15`
- FIR 계수 포맷: `signed 16-bit, Q1.15`
- 내부 곱셈 결과 포맷: `signed 32-bit, Q2.30`

비트 구조:

```text
[S . F F F F F F F F F F F F F F F]
 ↑   ↑
부호  소수부 15비트
```

- 표현 범위: `-1.0 ~ +0.999969482421875`
- 분해능: `2^-15 ~= 0.0000305`

### 5.2 확정 근거

1. 필터 계수와 자연스럽게 호환된다.
   - 계수 최대값이 `0.400133` 수준으로 `Q1.15` 범위 안에 충분히 들어온다.
   - 입력 `Q1.15`와 계수 `Q1.15`를 곱하면 내부 곱셈 결과를 `Q2.30`으로 해석할 수 있어 scaling 기준이 단순하다.
2. 기본 데모 신호 생성이 쉽다.
   - `numpy` 멀티톤을 `[-1.0, 1.0)` 범위로 정규화한 뒤 바로 `Q1.15`로 변환할 수 있다.
   - 예시:

```python
x_fixed = np.clip(
    np.round(signal * (2**15)),
    -(2**15),
    (2**15) - 1,
).astype(np.int16)
```

3. 확장 데모 ADC 경로와 호환된다.
   - 확장 데모에서 `PMOD AD1` 16-bit ADC를 사용할 경우, 입력 전압을 정규화하고 오프셋 제거 후 `[-1.0, 1.0)` 기준으로 매핑하면 `Q1.15`에 바로 연결할 수 있다.
4. 16-bit 양자화 SNR이 현재 스펙을 충분히 상회한다.
   - 16-bit 기준 양자화 SNR은 약 `96 dB`.
   - 현재 저지대역 감쇠 목표는 `As = 60 dB`.
   - 즉 `96 dB > 60 dB`이므로 입력 양자화 정밀도는 현 스펙 대비 충분하다.
5. 일반적인 DSP fixed-point 관행과도 맞는다.
   - 오디오/통신 계열의 `signed 16-bit` 정규화 신호 표현에서 `Q1.15`는 가장 흔한 기준 중 하나다.

### 5.3 선택 이유와 비선택 이유

- `Q1.15`를 선택한 이유:
  - 입력 신호가 정규화된 양/음 대칭 신호라는 가정과 자연스럽다.
  - 16-bit에서 fractional bit를 최대로 확보한다.
  - coefficient format과 통일되어 RTL/모델 설명이 단순해진다.
- `Q2.14`, `Q3.13` 등을 선택하지 않은 이유:
  - 정규화 입력 기준에서는 추가 정수 비트의 실익이 작다.
  - fractional bit 감소로 입력 정밀도만 낮아진다.

## 6. 검증 기준

- 동일 파라미터로 동일 입력이 재생성 가능해야 한다.
- alias 관찰 목적의 톤 배치가 문서에 설명되어 있어야 한다.
- ideal 검증과 fixed 동적 범위 분석에 필요한 신호 통계가 기록되어 있어야 한다.
- clipping 여부와 peak/RMS 값이 문서에 남아 있어야 한다.
