## 4. 해석 (sweep 진행하며 갱신)

**20000ps (느슨한 제약) — 잠정 "v1 ≈ v2"로 보였던 구간:**

- 두 버전 모두 critical path가 순수 탭 누산 경로(prod_reg[k]+z[k+1]→z_reg[k])의
  ripple-carry 체인(depth 85). 느슨한 제약에서 합성기가 면적 최소인 ripple 매핑을
  선택해 add 1회 자체가 지배적이었고, v2가 분할한 누산+라운딩 병합 경로는 한계를
  정하는 경로가 아니었다 → WNS 차이 1.4%p로 "분할은 FPGA 전용" 가설과 일치해 보임.

**15000ps (타이트한 제약) — 반전, 분할이 ASIC에서도 유효:**

- 제약을 조이자 합성기가 탭 가산기를 더 빠른 구조로 재합성(depth 85→50, 그 대가로
  cells +49%/area +19%). 그러자 **v1의 병목이 누산+라운딩 병합 경로
  (prod_reg→round_reg)로 이동 — FPGA에서와 같은, v2가 분할한 바로 그 경로다.**
- v2는 그 경로가 없어 WNS +4680.8 vs v1 +946.6 — **slack 5배 차이.** v2의 worst 경로는
  미제약 입력 경로(in_sample→prod_reg)라서 내부 reg-to-reg 한계는 이보다도 위에 있다.
- 잠정 Fmax 추정(1/(period−WNS)): v1 ≈ 71MHz, v2 ≥ 97MHz — **파이프라인 분할이
  ASIC에서도 유효하다는 방향.** "분할은 FPGA 전용 최적화" 가설은 느슨한 제약에서만
  성립하는 착시였고, 도구가 가산기를 최적화하고 나면 타겟 불문 "add+round 병합"이
  구조적 병목이라는 쪽으로 수렴 중. 확정은 pass/fail 경계(이분탐색)로.
- **전력·면적은 여전히 v1 ≈ v2** (ΔP −1.6%, Δarea +0.8%): 분할의 비용이 사실상 0이라는
  점도 v2의 손을 들어줌. leakage는 total의 0.001% — 250nm 예상대로 무시 가능,
  "ASIC total ≈ dynamic" 취급 유효.

**12000ps — 격차는 좁혀지나 순서는 유지, 공통 하한(곱셈 경로) 등장:**

- v1: 합성기가 라운딩 병합 경로를 더 조여(depth 50→40, cells +194) slack이 15000ps보다
  오히려 개선(+1133.5). v2: 15000ps netlist 그대로 재사용(여유 충분) — 한계는 입력→곱셈
  경로 arrival ~10039ps로 고정.
- 잠정 Fmax 추정: v1 ≈ 92MHz, v2 ≈ 97MHz — 15000ps에서 5배였던 slack 격차가 좁혀짐.
  ~~곱셈 경로 ~10ns가 공통 하한~~ → 10000ps에서 곱셈 경로도 재구조화되어 하한이
  아니었음이 확인됨(아래).

**10000ps — 수렴: 둘 다 PASS(100MHz), v1이 근소 우위로 역전:**

- v1 +199.0 vs v2 +129.2 (ΔWNS −69.8ps, period의 0.7%) — 12000ps까지의 v2 우위가
  사라지고 사실상 동률(오히려 v1이 근소 우위).
- 메커니즘: 합성기를 한계까지 밀자 **v1의 누산+라운딩 병합 경로가 9520.7ps까지 압축**됨
  (multi-operand add를 carry-save/compound 구조로 재조직 — 표준셀에서는 add 2회
  병합이 add 1회 + 압축단 수준의 비용). 한편 **v2는 자신의 라운딩 스테이지(z[0]→
  round_reg, 48-bit 라운딩 가산)가 새 병목**으로 등장, 9590.5ps — 결국 양쪽 다
  "라운딩이 포함된 최종 가산" 하나가 한계를 정하는 동일 구도로 수렴.
- **이것이 FPGA와의 구조적 차이**: FPGA CARRY4는 고정 캐리 체인이라 add 2회 병합을
  재구조화할 수 없어 v1의 병합 경로가 통째로 두 체인 값을 냈고(그래서 분할이 26%
  이득), ASIC 표준셀 합성은 그 병합을 흡수해버린다 — **"분할의 이득은 FPGA 물리
  구조에 종속" 가설이 pass/fail 경계 근처에서 결국 성립하는 방향.**
- 잠정 Fmax 추정: v1 ≈ 102MHz, v2 ≈ 101MHz. ~~8000ps는 확실한 FAIL~~ →
  9000ps 결과로 반증됨(아래) — 합성기의 스텝당 개선(~10%)이 아직 포화되지 않았다.

**9000ps — 둘 다 PASS(111MHz), v2 재역전, 순위는 런마다 요동:**

- v1 +100.2(margin 1.1%) vs v2 +526.3(5.8%) — 10000ps에서 뒤졌던 v2가 다시 앞섬.
  v2는 라운딩 스테이지가 재구조화로 물러나고 입력→곱셈 경로가 worst로 복귀
  (8193.4ps, 12000ps 대비 −18%). v1 병합 경로도 8619.5ps로 −9.5% 추가 압축.
- 순위가 런마다 뒤집히는 것 자체가 결론의 일부: **두 버전의 한계 경로 도달치가 수 %
  이내로 붙어 있어, 순서는 합성 휴리스틱의 런별 편차에 좌우된다.** FPGA의 26% 차이
  같은 구조적 격차는 ASIC에 없다.
- 잠정 Fmax 추정: v1 ≈ 112MHz, v2 ≈ 118MHz. **다음: 8000ps 페어** — v1은 −10.4% 추가
  압축이 필요해 경계 직전(직전 스텝 개선폭 −9.5%와 거의 같음), v2는 −5.8%면 됨.
  여기서 v1 FAIL / v2 PASS로 갈리면 그 지점이 최종 판별점.

**8000ps — 판별점에서도 안 갈림: 둘 다 PASS(125MHz), Δ 20ps로 완전 수렴:**

- v1 +52.0 vs v2 +72.3 — 차이 20.3ps는 period의 **0.25%**. v1 병합 경로 7667.7ps,
  v2 라운딩 스테이지 7647.4ps로 도달치까지 사실상 동일. "v1 FAIL/v2 PASS로 갈린다"는
  시나리오는 실현되지 않음 — v1의 병합 경로는 8000ps에서도 압축 여력을 찾아냄.
- margin 추이(2.0→1.1→0.7%)와 면적 증가(cells 25994→27359)를 보면 툴이 "겨우 맞추기"
  를 반복하며 한계에 접근 중. 잠정 Fmax: v1 ≈ 125.8MHz, v2 ≈ 126.1MHz — **격차 0.3%.**
- 결론 방향 사실상 확정: **ASIC(현대적 datapath 합성)에서는 v1/v2 구조적 격차가 없다.**
  v2 분할의 이득은 FPGA CARRY4 고정 체인에서만 발생하는 타겟 특화 효과.
- 다음: 아직 FAIL 앵커가 없음 — ~~7000ps~~ → 6000ps로 직행하기로 결정.

**6000ps — 여전히 둘 다 PASS(166.7MHz, v1은 slack +0.3ps 턱걸이). sweep 중단 결정:**

- −25% 압축 요구도 툴이 맞춰냄: v1 5718.7ps(slack +0.3), v2 5706.2ps(+13.5). Δslack
  13.2ps — 여전히 노이즈 수준 동률. **250nm ASIC이 28nm FPGA v2의 146MHz를 넘는
  166.7MHz에 도달**했다는 것도 부수 확인.
- v1의 critical path가 **FPGA v1과 시작·끝이 동일한 경로**(z_reg[1]→round_reg)로 회귀 —
  같은 경로를 FPGA CARRY4는 8.66ns(120MHz fail, log 39), 250nm 표준셀 재구조화는
  5.72ns에 처리. "툴이 병합을 흡수한다"의 가장 직접적인 대비 증거.
- **방법론적 교훈(중단 사유)**: margin이 매 지점 0으로 수렴하는 건 한계 신호가 아니라
  **제약 구동 합성의 본성**이다 — 툴은 목표를 주면 면적을 태워 "겨우 맞추기"를
  반복하므로(cells 25994→28615), pass/fail 경계 탐색은 진짜 물리 한계가 나올 때까지
  런당 수 시간짜리 시도를 계속 요구한다. 절대 Fmax 확정은 Track A 목적(보조 지표 +
  v1/v2 비교)에 불필요하므로 **여기서 중단, "양쪽 모두 ≥166.7MHz 확인(한계 미탐색),
  전 구간 격차 노이즈 수준"을 최종 기록으로 채택.**

**최종 결론 (sweep 종료):**

- 20000→6000ps 6개 페어 전 구간에서 v1/v2 격차는 최대 수 %·순위 요동 — **ASIC(현대적
  datapath 합성)에서 v1/v2 구조적 격차 없음.** v2 분할의 26% Fmax 이득은 FPGA CARRY4
  고정 캐리 체인에서만 발생하는 타겟 특화 효과로 확정.
- 면적·전력도 전 구간 Δ 1% 안팎 동률. 주파수에 따라 면적 +27%(1.12M→1.42M),
  전력 689→1331mW로 증가 — 제약을 조일수록 툴이 면적/전력을 태워 타이밍을 사는
  전형적 trade-off (GEMM sweep과 동일 패턴).
